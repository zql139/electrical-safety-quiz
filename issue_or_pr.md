# Issue 记录文档

## Issue #1: 题库分类不均衡问题

### 问题描述

经过数据分析发现，当前题库的15道题目在分类分布上存在不均衡现象：

| 分类 | 题目数量 | 占比 |
|------|---------|------|
| 用电安全相关 | 5题 | 33.3% |
| 火灾相关 | 3题 | 20% |
| 其他分类 | 7题 | 46.7% |

部分重要主题（如触电急救、雷电防护）只有1道题，而一些主题题目数量较多。

### 影响

1. **教学效果**：学生可能无法全面学习电气安全知识
2. **测试准确性**：随机抽题可能导致某些重要知识点被遗漏
3. **用户体验**：重复遇到相似类型的题目

### 建议解决方案

1. **增加题目数量**：将题库扩展到30-40道题
2. **均衡分类**：每个分类至少保证3道题目
3. **添加新分类**：如电气设备维护、安全标志识别等

### 优先级

**中** - 影响教学效果，但不影响核心功能

## Issue #2: 分数可能出现负数

### 问题描述

当前计分规则为：答对+1分，答错-1分。如果连续答错，分数可能变为负数。

### 影响

1. **用户体验**：负分可能打击学习积极性
2. **教学场景**：课堂环境中负分可能不合适

### 建议解决方案

```c
// 优化后的计分逻辑
if (userAnswer == quizQuestions[i].answer) {
    score += 1;
} else {
    score = (score > 0) ? score - 1 : 0;  // 最低分为0
}
```

### 优先级

**低** - 不影响核心功能，仅影响体验

## Issue #3: 选项顺序固定

### 问题描述

每道题的选项顺序是固定的，正确答案的位置也是固定的（多数在位置2或3）。

### 影响

1. **测试公平性**：熟悉题库的用户可能记住答案位置而非内容
2. **学习效果**：不利于真正掌握知识

### 建议解决方案

在显示题目时随机打乱选项顺序，但需要记录正确答案的对应关系。

### 优先级

**低** - 不影响核心功能
